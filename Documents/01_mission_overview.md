# 01. 미션 분석 (Mission Overview)

## 1. 대회 개요

- **대회**: 제 9회 국민대학교 자율주행 경진대회 (2026)
- **단계**: 예선 (Qualifier)
- **팀명**: Eri-car
- **환경**:
  - WSL2, Ubuntu 22.04, ROS2 Humble
  - 시뮬레이터: Windows 호스트에서 실행
  - 코드: WSL2 (Ubuntu) 내부에서 실행
  - 통신: TCP를 통해 시뮬레이터 ↔ ROS2 노드 데이터 교환
  - 워크스페이스: `~/xycar_ws`
  - ROS2 네임스페이스: `xycar`, Domain ID: 7

---

## 2. 미션 리스트

총 **3 lap** 주행하며, 다음 7가지 인식/판단 미션을 수행한다.

| # | 미션 | 설명 |
|---|------|------|
| 1 | 신호등 인식 | 시작 신호등(초록) 및 트랙 위 신호등(직진/좌회전 신호) |
| 2 | 라바콘 주행 | 출발 직후 라바콘 구간 통과 |
| 3 | 차선 인식 주행 | 1·2차선 주행 및 차선 변경, 차선 미검출 구간 포함 |
| 4 | 보행자 인식 | 위험 위치 보행자 감지 시 정지 및 재출발 |
| 5 | 경찰차 인식 | 경로 선택 지점에서 경찰차 유무 판단 |
| 6 | 방해차량 인식 | 1·2차선 차량 중 더 빠른 차량 판단 후 추월 |
| 7 | 어린이 보호구역 인식 | 시작 표시·종료 표시 노면 인식, 속도 제한 적용 |

## 3. 트랙 구조

### 핵심 지점 (트랙 다이어그램 기준)

```
                  ┌─ ⑥ 경로선택 신호등 (트랙 위 신호등) ──┐
                  │                                       │
   ┌──────────────┴─────────  큰 바퀴 (외곽) ─────────────┴──────┐
   │                                                                │
   │   ④ 보행자             ⑦ 종료                       ③ 아스팔트 │
   │     회피                                              진입     │
   │                                                                │
   │   ⑤ 방해차량                                        ② 라바콘   │
   │     추월                                              주행      │
   │                                                                │
   │              ⑥ 지름길 (Shortcut, 좌회전 시 진입)              │
   │                                                                │
   └────────────────────────  하단  ─────────  ① 신호등 인식 출발 ─┘
```

### 트랙 위 주요 요소

- **시작 신호등** (트랙 우하단): 출발 신호. 초록불 인식 시 출발
- **라바콘 구간** (트랙 우측): 출발 직후 한 번만 통과
- **트랙 위 신호등 = 경로선택 신호등** (중앙 상단): lap1에서는 직진 신호만 확인, lap2/lap3에서는 좌회전 신호 시 지름길 진입
- **경찰차** (경로선택 지점): 존재 시 지름길 차단, 큰 바퀴를 강제로 돌게 됨
- **보행자 구간** (좌측 상단)
- **방해차량 구간** (좌측 하단): 1·2차선에 차량 존재, 추월 필요
- **어린이 보호구역**: 지름길에서 빠져나오는 좌회전 직후, lap1의 경우 큰 바퀴 좌측 구간에 존재 (모든 lap에서 통과)
- **출발선 (체스판)**: lap 카운트 기준점, 경로선택 신호등 바로 앞에 위치
- **지름길 (Shortcut)**: 트랙 중앙을 가로지르는 길. 좌회전 두 번으로 진입/탈출

## 4. lap별 시나리오

### lap1: 큰 바퀴 (필수)

```
시작 신호등 대기 (대기)
  ↓ 초록불
라바콘 주행
  ↓ 라바콘 종료 + 차선 검출
차선 진입 후 트랙 위 신호등 확인
  ↓ 초록불 (빨강이면 정지 대기)
보행자 구간 통과 (필요 시 정지)
  ↓ 보행자 클리어
방해차량 구간 (속도 비교 → 추월)
  ↓ 추월 완료
어린이 보호구역 진입 (시작 표시 인식)
  ↓
어린이 보호구역 통과 (감속 주행)
  ↓ 종료 표시 인식
출발선 통과 → lap 카운트 +1
```

### lap2 / lap3: 경로 선택

lap 카운트 직후, **경로 선택 신호등** 앞에서 분기.

#### Case A. 경찰차 있음
- 신호등 빨강이면 정지, 초록불이면 직진
- 이후는 lap1과 동일한 큰 바퀴 흐름

#### Case B. 경찰차 없음
- 정지 후 좌회전 신호 대기 (이미 좌회전 신호가 떠 있으면 곧바로 좌회전)
- 첫 번째 좌회전 (지름길 진입, 목표 yaw₁)
- 지름길 내부 차선 주행 (일부 구간은 차선이 안 보임 → 차선 인식 모듈이 자체 대응)
- 두 번째 좌회전 (지름길 탈출, 목표 yaw₂)
- 좌회전 완료 직후 → 어린이 보호구역 진입
- 어린이 보호구역 통과
- 출발선 통과 → lap 카운트 +1

### lap == 3 종료
- 3 lap을 모두 마치고 출발선을 통과하면 시뮬레이션이 자동 종료된다.
- 별도 `FINISH` 스테이지를 만들지 않는다.

## 5. 주요 판단 항목 정리

차량이 자율적으로 판단해야 하는 항목은 다음과 같다.

| 판단 항목 | 판단 주체 | 비고 |
|----------|----------|------|
| 시작 신호등이 초록인가 | 인식 담당 | 한 번만 사용 |
| 트랙 위 신호등이 초록/좌회전/빨강인가 | 인식 담당 | lap마다 활용 방식 다름 |
| 보행자가 지나가도 괜찮은 위치인가 | 인식 담당 | 단순화: 위험하면 true |
| 1·2차선 중 어떤 차량이 더 빠른가 | 인식 담당 | 일정 시간 관찰 |
| 지금 차선 변경/추월 시점인가 | 인식 담당 + main | 인식이 신호 → main이 결정 |
| 어린이 보호구역 시작/종료 | 인식 담당 | 노면 색상/표시 기반 |
| 경찰차 유무 | 인식 담당 | 경로선택 직전에 판단 |
| 지름길 두 번째 좌회전 트리거 | 인식 담당 | 차선 패턴(끝남) 등 |
| 출발선 통과 (lap 카운트) | 인식 담당 → main이 카운트 | 체스판 무늬 |
